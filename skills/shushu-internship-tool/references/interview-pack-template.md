# Interview Pack Template

## Profile Header

面试包开头先写清楚：

- 目标岗位：
- 用户水平：
- 技术栈偏好：
- 时间预算：
- 资源条件：
- 运行深度：`interview-only` / `smoke-test` / `local-full-run` / `remote-full-run`
- 当前项目状态：未 clone / 已 clone / 已跑 smoke test / 已完整跑通 / 已改造

## STAR 简历项目

写 4-5 行，尽量短：

1. S/T：项目背景和目标岗位问题。
2. A：你如何跑通最小路径并摸清核心链路。
3. A：你做了什么可面试的改造或工程整理。
4. R：有指标写指标；没有指标写 demo、报告、实验设计或代码产出。
5. R：产物，如 demo、报告、可复现脚本或面试展示。

禁用写法：

- “熟练掌握”但没有具体模块。
- “显著提升”但完全没有指标或对比口径。
- “独立完成某某系统”但面试里讲不清哪些是开源基础、哪些是自己的理解和调整。

## 面试官拷问

至少覆盖：

- 项目为什么匹配这个 JD？
- 你选择的运行深度是什么？为什么当前阶段不完整跑/为什么需要完整跑？
- 原项目原始能力是什么？你重点理解和调整了什么？
- 输入输出是什么？请求字段、页面状态、数据库表、消息格式、文件格式或张量 shape 是什么？
- 主链路入口在哪里？API、页面、CLI、worker、定时任务、训练脚本或部署脚本分别怎么走？
- 关键技术为什么这样选？框架、数据库、缓存、队列、状态管理、测试策略、模型或 metric 怎么取舍？
- 最小路径怎么跑，失败怎么排查？
- 指标或结果怎么解释？如果没完整跑完，下一步怎么验证？
- 如果线上部署，延迟、吞吐、成本、失败恢复怎么考虑？
- 最大限制是什么？下一步会怎么做？

## 核心代码讲解

讲解顺序：

1. 入口命令。
2. 配置/参数。
3. 核心输入输出：HTTP request、页面事件、DB row、message、file、dataset 或 tensor。
4. 核心模块：controller/service/repository/component/store/job/model 等。
5. 关键状态变化：数据库、缓存、队列、页面状态、任务状态或模型输出。
6. 测试、评估、日志、监控或指标。
7. 你的改动文件。
8. 一个失败 case 或 debug 过程。

## PPT 提示词

让模型生成 PPT 时必须加入：

- 目标岗位和项目匹配。
- 项目来源和你选择它的理由。
- 架构图、请求流图、状态流图、任务流图或数据流图。
- baseline 命令、smoke test 或计划。
- 当前运行深度和下一步执行计划。
- 我的修改、理解和面试亮点。
- 指标表；没有指标时改成实验计划、工程产出和失败定位。
- 限制与后续计划。

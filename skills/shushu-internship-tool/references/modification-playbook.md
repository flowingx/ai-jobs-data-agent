# Modification Playbook

选择魔改点时优先“小而硬”：推进快、能解释、能写进简历、能抗住追问。

## 低风险改造

- 后端：新增一个 CRUD/API、分页/搜索、参数校验、统一错误码、OpenAPI 文档。
- 前端/移动端：新增页面、表单校验、loading/error/empty 状态、响应式布局、接口联调。
- 全栈：打通一个端到端业务流，例如注册登录、工单创建、订单查询、文件上传。
- 测试：补单测、接口测试、mock 数据、CI smoke test。
- 数据：加数据清洗脚本、指标 SQL、简单 dashboard 或数据质量检查。
- DevOps：加 Docker Compose、启动脚本、健康检查、基础日志。
- 安全：加输入校验、权限检查、敏感配置读取、依赖版本检查。
- AI/算法：换小数据集、加 demo、加 evaluation table、加 error analysis。

验收：本地可运行或能清楚解释卡点，有 before/after 说明和面试话术。

## 中风险改造

- 后端：加 Redis 缓存、消息队列、异步任务、RBAC、数据库迁移、限流。
- 前端/移动端：加状态管理、权限路由、组件抽象、图表可视化、离线缓存。
- 全栈：前后端联调、部署到云服务器、加 demo 数据和一键启动。
- 测试：加 E2E、覆盖率、契约测试、压测和性能报告。
- 数据：加调度、增量同步、schema 演进、数据血缘或指标口径文档。
- DevOps：加 CI/CD、Kubernetes manifest、日志收集、监控面板。
- 安全：加审计日志、鉴权中间件、依赖扫描、常见漏洞修复说明。
- AI/算法：换 backbone/embedding、加 ablation、加吞吐/延迟 benchmark。

验收：至少一个 baseline 或 smoke test，加上 modified 设计；有指标最好，没有指标就准备工程效果和下一步实验计划。

## 高风险改造

- 后端/系统：重构核心架构、分库分表、分布式事务、高并发网关、存储引擎。
- 前端/移动端：重写状态架构、微前端、跨端方案、复杂性能优化。
- 数据：流批一体、实时数仓、湖仓、复杂 DAG 调度和容灾。
- DevOps：完整 GitOps、多环境发布、服务网格、弹性伸缩、成本优化。
- 安全：权限模型重构、攻防验证、沙箱、供应链安全。
- AI/算法：换任务或跨数据域迁移，引入 LoRA、distillation、self-training、MLOps 监控。

验收：先做 smoke test，再小样本实验，最后才考虑完整训练。面试中要主动讲限制和失败 case。

## 不推荐改造

- 只改 README、变量名、颜色、标题。
- 只把别人的结果截图搬进报告，自己完全讲不出流程。
- 没跑通 baseline 就硬写具体提升百分比。
- 引入自己解释不清的复杂库、云服务或大模型服务。

## 选择规则

- 时间少：低风险改造 + 强代码讲解 + 面试 Q&A。
- 有云服务器或 Docker：中风险工程改造 + 一键启动/demo + 对比说明。
- 有单卡 GPU：AI/算法中风险改造 + 小样本 baseline + 对比表。
- 已有相关基础：做一个高风险探索，但简历表达优先写设计、实现和工程产出。
